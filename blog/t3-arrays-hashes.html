Arrays vs. Hashes, Hashes vs. Arrays
13 November 2014

Array vs. Hash, Hash vs. Array. Which one is better? How are they similar? How are they different? Well, I will tell you the short version of everything you will need to know to uses either one of these efficiently. I could give you a technical definition but the simple version is that they both store data. That's it, they keep your information in a nice easily accessible place. But why are there two? Why do we need both?

I know these two might be a little confusing but with real example you will see that they are straight forward. The simple way of looking at the difference between an array and a hash is to actually show you what they look like and how they work.

An array: is an ordered list of things.
An array looks like this: my_array = [a, b, c, d].

An array stores anything you put in it, it is a list of anything you would like. For example is you want to store your favorite sport teams; you can have an array like:

favorite_teams = [Giants, 49ers, Kings]

On the contrary, a hash is just a fancier version of an array. A hash: is a collection of key/value pairs. A hash looks like this:

my_hash = {a => "w", b => "x", c=> "y", d => "z"}

In the same example as above, if you would like to add the sport in which your favorite team plays you can have a hash like:

favorite_teams = {baseball => "Giants", football => "49ers", basketball => "Kings"}

Where you can put any values into the variables. Easy enough, a hash points an object to another object. A key and a value. Think of it putting an answer the question. The key as the question and the value as the answer. In a real life example, you know when you buy something online and you have to type in all of your info, like: name, address, zip, state and so on. A ruby website might use a hash to store all of this data. This is a little easier way of reading a hash.

my_info = {
name => "Megan Fox",
city => "Oak Ridge",
state => "Tennessee",
zip => "37830",
}

Well that's it. Arrays and Hashes. Hashes and Arrays. I hope now you can successfully use each one of these. Once you get these down, it is much easier to store your data.
