Relative, Absolute, and Fixed Positioning
5 November 2014
What are the differences between relative, absolute, and fixed positioning?

Before we dive into the relative, absolute, and fixed positioning. We need to learn more about the background on the subject. First, we need to wrap our heads around the fact that every single element on a webpage is a block, meaning everything in a webpage is literally rectangles of pixels. And how these blocks of rectangles are positioned can be static, relative, absolute, or fixed positioning. Before you add any coding or styles to the element, the default positioning is called static. Everything, in every page starts out as static. Static means that the elements will flow into the page just as it normally would. Static positioning is the very basic and is not affected by top, bottom, left, and right attributes.

Relative position: relative

A relative position element is positioned relative to its normal position. Ultimately if you set a relative position to an element without any other positioning attributes, it will work exactly the same as the default, static. However, if you do give it some positioning, like left: 20px and top: 10px, it will shift its position 20 pixels right and 10 pixels down from where it was originally assigned. This technique is nifty as it then makes it easier to manually shift elements around just by assigning them their own positioning attributes.

Absolute position: absolute

The technical term for absolute is: an absolute positioned element is positioned relative to the first parent element that has a position other than static. Although absolute can be neat, absolute can possibly be damaging to you site as it can possibly limit its flexibility.

Fixed position: fixed

Unlike absolute positioning, where it is positioned to the relative to the first parent element, a fixed positioned element is positioned relative to the viewpoint or browser window. Even if you scroll down or across the page the element will remain exactly where is it not matter how far you scroll. For example, a fixed positioned element can be compared to that bar at the top of Facebook, since no matter how far down the page you scroll, you will always see it. Fixed elements can be good or bad, they can allow you to keep the navigation of your site accessible at all times, however often times with smaller screens the element can overlap other features and if used wrong it will diminish your user experience.

All of these positioning have their use in modern browsers, however deciding when to use each one of these can be troubling. No matter how you decide to use these tools, remember that the user accessibility and user experience is the true end result.
