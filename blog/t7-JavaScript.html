Ruby vs. JavaScript
9 December 2014

Well, I have to say that I love Ruby. Ruby seems to be such as easy language to understand and you have the ability to create programs that people without a computer background can actually understand. I have been working with Ruby and for the past month or so and I have now just launched myself into JavaScript and I have to say that other than the different syntaxes, these object oriented languages are very similar. I could go on and on about all of the different similar things that these two language share, but in this post, I am going to focus on Ruby classes vs. JavaScript constructor functions. So how are Ruby Classes and JavaScript functions different and how are they similar? First you can kind of think of functions in JavaScript similar to methods in Ruby. The syntax of declaring a function vs. declaring a method is slightly different. In Ruby:

def hello
puts “hello”
end

In JavaScript, the same logic can be written:

function hello() {
console.log(“hello”); }

Both these print the word “hello” to the screen in their respective languages. And as you can see they have their similarities and their differences. Simplified this works but what happens when we get a little more advanced.

So back to the basics. In Ruby, classes are objects and the unique thing about Ruby classes are you can easily define new instances in Ruby with a special Ruby method called initialize. In JavaScript, objects don’t have formal classes, and don’t get the use of an initialize like in Ruby. JavaScript instances are created with something called a constructor. This constructor of an instance is a function that is called with the new operator. The cool thing about JS is that any function can become a constructor. Here is an example:

function square (n) { return n * n; }
//=> undefined
square(2)
//=> 4
square(2).constructor
//=> [Function: Number]
new square(2)
//=> {}
new square(2).constructor
//=> [Function: square]

In this example the square function will act as a constructor if you call it with new. This aspect is special in JavaScript. JavaScript also has something called prototype. These prototypes are their way of defining instances of classes like in Ruby. Here is an example of the difference between the two languages.

Ruby

class Person
attr_accessor :firstName, :lastName

def initialize(firstName, lastName)
@firstName = firstName
@lastName = lastName
end

def fullName
@firstName + " " + @lastName
end
end

john = Person.new("Babe", "Ruth")
john.fullName # "Babe Ruth"

function Person(firstName, lastName) {
this.firstName = firstName;
this.lastName = lastName;
}

Person.prototype.fullName = function() {
return this.firstName + " " + this.lastName;
}

var john = new Person("Babe", "Ruth");
john.fullName(); // "Babe Ruth"

As you can see, they both output “fulName.” Now you can decide which language do you like better, Ruby or JavaScript?
