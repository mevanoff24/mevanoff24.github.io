Variable Scope
2 December 2014

Time for one of the most important things in Ruby. Drumroll Please. Dadadadadadadadadadadadadada. Variable Scope! (Cheers throughout the crowd!) I know they are just that cool. Ok, you got me, maybe not that cool, but variables are extremely important to any program as can make your life a living hell if you don’t know how to use them. So it is best to learn them now.

Alright, back to the beginning, what is a variable?

A variable is simple a place to store data. A variable holds objects in the computer’s memory. One of the cool things is that Ruby belongs to the family of dynamic languages and variables can contain different values and different types of values over time. A basic example of this would be:

x = 3
puts x
x = 5
puts x
This would return
3
5

Cool right? You can also increment “x” by one or many more possibilities. Therefore if you want to keep you variable constant you can and if you need to continue to change the variable data you can as well.

Ok, back to variable scope. A variable is always given a unique name. Each unique name has a unique reason. Ruby has four types of variable scopes, local, global, instance, and class. As well, as one constant type. The cool and easy thing about variable scope in Ruby is that each variable type is declared by using a special character at the start of the variable name. In addition, Ruby has two pseudo-variables which cannot be assigned values. These are nil which is assigned to uninitialized variables and self which refers to the currently executing object.

Local Variables

Local variables are variables that are valid within a local area of a Ruby source code. Also referred to as local scope. A local variable declared in a method cannot be accessed outside of that method. Local variable names must begin with either an underscore or a lower case letter. For example:

local_variabele = “Hello World”

Global Variables

Global variables in Ruby are accessible from anywhere in the Ruby program, regardless of where they are declared. Variable identifiers can start with special characters also called sigils. A sigil is a symbol attached to an identifier. The Ruby variable sigils are $ and @. In this case, Global variable names are prefixed with a dollar sign ($). For example:

$global_variable = “Hello World”

However, the use of global variables is highly discouraged. Global variables tend to lead to programming errors and global variables should be used only when you have to.

Class Variables

Another way of thinking of class variables is as global variables within the context of a single class. A class variable is shared amongst all instances of a class. This means that only one variable value exists for all objects instantiated from this class. Class variables also use sigils and are declared by prefixing the variable name with two @ characters (@@). For example:

@@class_variable = “Hello World”

Instance Variables

Instance variables are similar to Class variables except that their values are local to specific instances of an object. Instance variables are declared in Ruby by prefixing the variable name with a single @ sign. For example:

@instance_variable = “Hello World”

Constant Variables

Constant Variable, once assigned a value, should not be changed. Constants declared within a class or module are available anywhere within the context of that class or module. Constants declared outside of a class or module are assigned as global scope.

Pseudo Variables

Pseudo variables are different from regular variables. We cannot assign values to pseudo variables. The self is the receiver of the current method. The nil represents the absence of a value. These variables are part of the variable scope however are used differently than the other four normal variables.

So variable scope isn’t as bad as you think. But once you master these variables you are well on your way to becoming a Ruby master.
